\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysection{\texorpdfstring{Visão geral}{Visão geral}}\label{md__r_e_a_d_m_e_autotoc_md4}
O canal transmite frames Ethernet broadcast de {\bfseries{270 bytes}} com Ether\+Type {\ttfamily 0x88\+B5} (IEEE private use, sem significado IANA). Cada frame carrega um payload de 256 bytes ({\ttfamily \doxylink{structstealth__pkt__t}{stealth\+\_\+pkt\+\_\+t}}) com magic {\ttfamily "{}STLH"{}}, número de sequência, IV aleatório, ciphertext AES-\/256-\/\+CTR e tag HMAC-\/\+SHA256.

O programa XDP ({\ttfamily \doxylink{xdp__kern_8c}{xdp\+\_\+kern.\+c}}), carregado em modo native DRV no driver {\ttfamily mlx5}, filtra os frames por Ether\+Type, valida o magic, aplica anti-\/replay via {\ttfamily replay\+\_\+map} (LRU hash), e redireciona os frames para o socket AF\+\_\+\+XDP via {\ttfamily bpf\+\_\+redirect\+\_\+map}. Em hosts sem o handler XDP, os frames são silenciosamente descartados pelo kernel como Ether\+Type desconhecido.

O MAC de origem é gerado com {\ttfamily fread(/\+dev/\+urandom)}, com o bit locally-\/administered ({\ttfamily mac\+[0]\+ \texorpdfstring{$\vert$}{|}= 0x02}) e o bit multicast limpo ({\ttfamily mac\+[0]\+ \&= 0x\+FE}), resultando num endereço unicast válido mas diferente por frame — impede correlação de frames por análise L2.\hypertarget{md__r_e_a_d_m_e_autotoc_md18}{}\doxysubsection{\texorpdfstring{Camada 2 — XDP Interception}{Camada 2 — XDP Interception}}\label{md__r_e_a_d_m_e_autotoc_md18}
O programa BPF é carregado via {\ttfamily bpf\+\_\+object\+\_\+\+\_\+open} + {\ttfamily bpf\+\_\+object\+\_\+\+\_\+load} e anexado com {\ttfamily bpf\+\_\+xdp\+\_\+attach(..., XDP\+\_\+\+FLAGS\+\_\+\+DRV\+\_\+\+MODE, NULL)}. Antes do attach, são feitos detach de todos os modos (DRV, SKB, HW) para eliminar programas residuais de sessões anteriores, seguido de {\ttfamily usleep(100ms)} para o kernel processar.

Após o attach, o daemon verifica que o {\ttfamily prog\+\_\+id} do programa anexado à interface corresponde ao {\ttfamily prog\+\_\+id} do objeto carregado — falha explicitamente se houver mismatch.

O socket AF\+\_\+\+XDP é criado com {\ttfamily XSK\+\_\+\+LIBBPF\+\_\+\+FLAGS\+\_\+\+\_\+\+INHIBIT\+\_\+\+PROG\+\_\+\+LOAD} (não carrega um segundo programa XDP) e registado em {\ttfamily xsks\+\_\+map\+[0]\+} via {\ttfamily bpf\+\_\+map\+\_\+update\+\_\+elem}.\hypertarget{md__r_e_a_d_m_e_autotoc_md19}{}\doxysubsection{\texorpdfstring{Camada 3 — AF\+\_\+\+XDP Userspace Consumer}{Camada 3 — AF\+\_\+\+XDP Userspace Consumer}}\label{md__r_e_a_d_m_e_autotoc_md19}
