Descrição detalhada

Programa eBPF executado no Kernel para filtragem e redirecionamento de pacotes.

Este módulo implementa a filtragem em Linha de Comando (XDP fast-path) para um canal encoberto de Camada 2. Valida o EtherType customizado, inspeciona bytes mágicos, protege contra ataques de replay via tabelas Hash LRU e redireciona o tráfego legítimo para o socket AF_XDP.

Mapa eBPF Per-CPU Array para Métricas e Estatísticas.

Mapa eBPF do tipo LRU Hash para Proteção contra Replay.

Faz a associação entre o índice da fila da placa de rede (Queue Index) e o File Descriptor (FD) do socket AF_XDP registado em userspace.

Armazena temporariamente os números de sequência (seq) já processados. Utiliza a política Least Recently Used (LRU) para descartar registos antigos quando atinge o limite máximo de 65.536 chaves, prevenindo exaustão de memória.

Vetor indexado para auditoria em runtime sem contenção de locks entre núcleos:

  • Índice 0: Contagem de pacotes válidos aceites e redirecionados.
  • Índice 1: Contagem de pacotes duplicados/rejeitados por replay.

◆ xdp_stealth_filter()

int xdp_stealth_filter ( struct xdp_md * ctx)

Filtro eBPF principal anexado ao gancho (hook) XDP do driver de rede.

Inspeciona sequencialmente o frame recebido a nível de hardware:

  1. Verifica se o tamanho do frame comporta o cabeçalho Ethernet standard.
  2. Filtra pelo EtherType customizado 0x88B5 (repassa o resto ao SO via XDP_PASS).
  3. Inspeciona a assinatura mágica no cabeçalho stealth.
  4. Valida se o número de sequência já foi visto no mapa LRU para evitar Replay (descarta via XDP_DROP).
  5. Efetua o bypass da stack de rede injetando o pacote na Queue 0 do socket AF_XDP.
    Parâmetros
    ctxPonteiro para os metadados contextuais do frame de rede (xdp_md).
    Retorna
    Decisão de ação XDP (XDP_PASS, XDP_DROP ou resultado de bpf_redirect_map).
